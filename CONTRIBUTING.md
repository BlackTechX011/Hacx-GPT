## Code of Conduct

This project is intended to be a safe, welcoming space for collaboration, and contributors are expected to adhere to the code of conduct.

## How Can I Contribute?

### Reporting Bugs

Start by searching GitHub Issues to make sure the bug hasn't been reported yet. If it hasn't, open a new issue using the Bug Report template.

### Suggesting Enhancements

If you have a suggestion for the project, we'd love to hear it. Open a new issue using the Feature Request template.

### Pull Requests

1. Fork the repo and create your branch from `main`.
2. If you've added code that should be tested, add tests.
3. Ensure the test suite passes.
4. Make sure your code lints.
5. Issue that pull request!

## Style Guide

- Use Python's PEP 8 style guide.
- Use `rich` for all terminal output.
- Keep the "Hacker" aesthetic in mind for UI changes.
